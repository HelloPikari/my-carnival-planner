# Session: Architecture Rethink
**Date:** 2026-05-25
**Participants:** Steve + Claude

## What Happened

Full architecture rethink via a grilling session. Threw out the original plan (Hono standalone API on DigitalOcean) and arrived at a new, clearer product and tech vision.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Switch from Hono → Next.js (App Router) on Vercel | Core product is a consumer web app, not an API-as-a-product. Next.js unifies the web app, API routes, and MCP server in one deployment. |
| WorkOS for all users (consumers + admin) | One auth system. MAU pricing is fine at this stage. AuthKit's hosted UI handles consumer signup. |
| Tiers: Free / Pro / Premium | Free = limited browse, Pro = full content + itinerary + groups, Premium = Pro + AI assistant access |
| MCP server = admin tool now, Premium tier later | Phase 3 builds it for Steve + Nicole (content management). Phase 5 exposes it to Premium subscribers. |
| Nicole is business partner, not just a user | She curates data via the MCP server (read + write). She also uses it for business ops queries. |
| Convert this repo to Next.js (don't start a new one) | Drizzle schema stays, git history stays. Add Next.js App Router alongside existing `src/db/`. |
| DB stays on DigitalOcean Postgres | Keep existing seeded data. Vercel Postgres (Neon) is unnecessary cost. |
| User schema needs updating | Add `workosId`, `subscriptionPlan`, `subscriptionStatus`. Remove `passwordHash` and `sessions` table (WorkOS handles tokens). |

## Architecture Summary

```
Next.js App (Vercel)
├── app/                    → Pages + route handlers
├── app/api/mcp/[transport] → MCP server (mcp-handler + withMcpAuth)
├── app/api/oauth/          → WorkOS OAuth proxy routes
├── app/.well-known/        → RFC 8414 + RFC 9728 discovery endpoints
└── src/db/                 → Drizzle schema + migrations (unchanged)
         ↓
Postgres (DigitalOcean)     → Existing seeded data
         ↑
WorkOS                      → Auth for all users
```
