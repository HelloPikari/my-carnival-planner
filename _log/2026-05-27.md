## What Happened

### nginx SSL → PostgreSQL SSL fix
- Root cause: nginx stream `listen 5433 ssl` is incompatible with PostgreSQL's SSL negotiation protocol (PostgreSQL sends an 8-byte SSLRequest before the TLS handshake; nginx expects TLS ClientHello as the first bytes)
- Fix: removed SSL from nginx stream config (plain TCP proxy), configured Postgres SSL natively with the Let's Encrypt cert for `db.mycarnivalplanner.app`
- Cert copied to `/var/lib/postgresql/16/main/server.crt` + `server.key`, renewal hook created at `/etc/letsencrypt/renewal-hooks/deploy/postgres-cert.sh`
- Added `hostssl` rule to pg_hba.conf for `mcp_db` to enforce SSL on remote connections

### Production DATABASE_URL
```
postgres://mcp_db:PASSWORD@db.mycarnivalplanner.app:5433/mcp_db?sslmode=require
```
Note: `sslrootcert=system` is NOT supported by postgres.js (causes "unrecognized configuration parameter" FATAL). Use `sslmode=require` only.

### Vercel deploy issues fixed
1. **Repo public** — Hobby plan blocked deploy from private repo when commit author wasn't recognized; made repo public

4. **`sslrootcert=system` in DATABASE_URL** — postgres.js forwards unknown URL params as Postgres SET commands; removed it
5. **`signOut` type mismatch** — wrapped in inline `"use server"` function to satisfy form action type
